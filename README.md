# 图书管理系统

复旦大学数据库课程中期实验——基于 Flask 的图书管理系统

## 项目简介
一个功能完整的图书管理系统，支持图书管理、进货管理、销售管理、财务统计、用户管理等功能。前端使用纯HTML+CSS+JavaScript（含Chart.js可视化图表），后端使用Flask+SQLite数据库。

## 功能模块

### 用户管理
- 超级管理员可添加/修改/删除普通管理员
- 支持 CSV 批量导入用户
- 用户角色：超级管理员、普通管理员
- 个人中心：修改个人信息、修改密码

### 图书管理
- 图书增删改查
- 支持 ISBN、书名、作者、出版社、价格区间等多种查询方式
- 模糊搜索
- CSV 批量导入/导出

### 进销存管理
- **进货管理**：创建进货单 → 付款 → 入库（完整流程）
- **销售管理**：录入销售，自动扣减库存
- **库存管理**：实时查看库存，低库存预警（≤5 本标红）
- 进货支持新书进货和已有图书进货

### 财务统计
- 自动记录所有收支账单
- 月度收支趋势图（柱状图 + 折线图）
- 收支占比饼图
- 图书销售排行榜（金额/数量），支持水平柱状图
- 支持按时间段筛选

### 操作日志
- 记录所有用户的操作行为
- 支持按用户名、操作类型、时间范围筛选
- 普通管理员仅查看自己的日志，超级管理员可查看全部

## 技术栈

| 技术 | 用途 |
|------|------|
| **Python 3** | 后端语言 |
| **Flask** | Web 框架 |
| **SQLite** | 数据库 |
| **Pillow / Chart.js** | 数据可视化 |
| **HTML + CSS + JS** | 前端界面 |

## 运行项目

### 1. 安装依赖

```bash
pip install flask
```

### 3. 启动系统

```bash
python app.py
```

默认访问地址：`http://localhost:5000`

### 4. 默认管理员账号

| 用户名 | 密码 | 角色 |
|--------|------|------|
| `admin` | `123456` | 超级管理员 |

登录后可创建更多普通管理员账号。

## 数据库表结构

| 表名 | 说明 |
|------|------|
| `user` | 用户信息（含角色权限） |
| `book` | 图书信息（ISBN、书名、作者、出版社、零售价） |
| `inventory` | 库存信息 |
| `purchase` | 进货单（含状态：未付款→已付款→已入库/已退货） |
| `sale` | 销售记录 |
| `bill` | 财务账单（收入/支出） |
| `log` | 操作日志 |

## 项目结构

```
├── app.py          # Flask 主应用（路由 + 页面渲染）
├── database.py     # 数据库连接与初始化
├── user.py         # 用户管理逻辑
├── book.py         # 图书管理逻辑
├── purchase.py     # 进货管理逻辑
├── sale.py         # 销售管理逻辑
├── bill.py         # 账单管理逻辑
├── log.py          # 日志管理逻辑
├── sample_books.csv   # 示例图书数据
├── sample_users.csv   # 示例用户数据
├── bookstore.db       # SQLite 数据库文件（运行后自动生成）
└── README.md
```
